## 摘要
基于文本条件的三维生成在图像和孤立物体方面发展迅速，但生成手物网格仍具挑战：输出必须保持语言语义、跨视图一致性、物体几何、关节手部形状以及物理合理的接触。我们提出TextHOI-3D，一种分阶段框架，利用生成的多视图观测作为文本条件视觉生成与几何感知手物恢复之间的显式接口。TextHOI-3D学习固定相机手物观测的紧凑VQ令牌空间，通过CLIP条件视觉自回归模型从文本预测多视图视觉令牌，并通过先验初始化、多视图联合优化和抗穿透细化恢复统一的手物网格。该设计将语义生成与几何恢复分离，同时通过离散多视图表示连接两个阶段。在HO3D衍生评估中，与单视图对应方法相比，多视图设置将物体CD从17.26毫米降至4.92毫米，穿透体积从5.3721立方厘米降至0.2193立方厘米，同时改进了手部误差和表面F分数。这些结果支持多视图视觉令牌作为文本驱动三维手物网格创建的有效中间表示。

## Abstract
Text-conditioned 3D generation has progressed rapidly for images and isolated objects, but producing a hand-object mesh remains challenging: the output must preserve language semantics, cross-view consistency, object geometry, articulated hand shape, and physically plausible contact. We present TextHOI-3D, a staged framework that uses generated multi-view observations as an explicit interface between text-conditioned visual generation and geometry-aware hand-object recovery. TextHOI-3D learns a compact VQ token space for fixed-camera hand-object observations, predicts multi-view visual tokens from text with a CLIP-conditioned visual autoregressive model, and recovers a unified hand-object mesh through prior initialization, multi-view joint optimization, and anti-penetration refinement. The design separates semantic generation from geometric recovery while keeping both stages connected by a discrete multi-view representation. On HO3D-derived evaluations, the multi-view setting reduces object CD from 17.26 mm to 4.92 mm and penetration volume from 5.3721 cm^3 to 0.2193 cm^3 compared with a single-view counterpart, while improving hand errors and surface F-scores. These results support multi-view visual tokens as an effective intermediate representation for text-driven 3D hand-object mesh creation.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：文本条件3D生成在图像和孤立物体上进展迅速，但生成手与物体交互的网格（hand-object mesh）仍非常困难。一个合理的结果必须同时满足语言语义保持、跨视图一致性、物体几何正确、手部关节形状准确以及物理上合理的接触（避免漂浮或穿透）。现有直接文本到网格的方法缺乏足够的配对数据，且手物接触区域常被遮挡。
- **整体含义**：提出一种分阶段替代方案：先由文本生成连贯的多视图视觉观察（图像），再从中恢复统一的3D手物网格。多视图图像比网格更容易合成，同时保留了轮廓、遮挡模式和接触线索，可作为生成与几何恢复之间的中间表示。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将文本到3D手物交互分解为三个技术阶段：离散多视图表示学习、文本条件多视图生成、联合网格恢复。通过离散VQ token作为紧凑的接口，分离语义生成与几何恢复。
- **关键技术细节**：
  1. **离散多视图视觉表示**：使用VQ-VAE，将固定相机布局下的N个RGB视图沿通道维度堆叠成H×W×3N的栈张量，由编码器映射为16×16的连续特征，通过包含4096个条目的码本量化，再解码恢复视图。训练损失包含重构损失、码本承诺损失等。
  2. **文本条件多视图生成**：采用Visual Autoregressive (VAR) 模型的“下一尺度预测”范式，从粗到细生成多尺度token金字塔。CLIP文本特征通过全局AdaLN调制（稳定全局语义）和局部交叉注意力（对齐局部文本属性）注入。训练使用教师强制，推理使用classifier-free guidance和top-k/top-p采样。
  3. **手物网格恢复**：生成的多个视图经过HSV分割、Stable Diffusion LoRA修复（处理遮挡），然后分别用InstantMesh初始化物体、用OmniHands风格手部先验初始化手部，再通过多视图联合优化（含重投影、掩码、接触、穿透惩罚项）以及抗穿透后处理，得到统一网格。优化变量包括手部参数、手部变换、物体位姿、物体形变修正。
- **公式/算法流程**：
  - VQ-VAE损失：\( L_{VQ} = \|X - \hat{X}\|_2^2 + \|sg[z_e] - z_q\|_2^2 + \beta \|z_e - sg[z_q]\|_2^2 \)
  - 生成模型：\( p(R|y) = \prod_{k=1}^{K} p(r_k | r_{<k}, y) \)
  - 恢复目标：\( \Xi^* = \arg\min_{\Xi} \lambda_{mv} L_{mv} + \lambda_c L_{contact} + \lambda_p L_{pen} + \lambda_r L_{reg} \)，其中 \( L_{contact} \) 鼓励手部接触区域靠近物体表面，\( L_{pen} \) 惩罚手部顶点进入物体内部。

## 7. 优点

- **紧凑高效的表示**：VQ token空间（16×16，4096码本）压缩了多视图信息，避免直接生成像素，提升生成效率。
- **文本控制精细**：CLIP条件结合全局AdaLN和局部交叉注意力，能同时控制交互语义和局部属性（如手柄、抓握）。
- **恢复流程鲁棒**：集成分割、修复、多个先验（物体InstantMesh、手部OmniHands/MANO）、多视图优化和抗穿透细化，解决遮挡和物理合理性。
